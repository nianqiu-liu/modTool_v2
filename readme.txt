一、软件名称：[game]mod管理工具,其中[game]部分为config文件中的game字段

二、文件说明
    2.1：配置文件，文件名为'config.txt'
        位置：软件同目录下
        内容共三行：
            第一行为[game]字段，表示游戏的名称
            第二行为[mod_dir]字段，表示可用mod的源路径
            第三行为[game_dir]，表示mod需要放置的游戏目标路径
    2.2：[mod_dir]文件夹，存放所有可用mod
        其结构及说明如下：
            [mod_dir]，mod的根目录
                |-[mod_name]，具体mod的文件夹，文件夹名称为[mod_name]，可有多个，不可重名
                    |-[预览图]，所在子文件夹对应的mod的预览图，可以有多张
                    |-[mod文件]，所在子文件夹对应的mod的具体mod文件，可以有多个
        文件名规则：
            文件名包含了内容层级，用‘-’分割，比如假设有三个mod文件夹，分别为'角色a-服装1'，'角色a-服装2','角色b-服装1',则表示共有两个一级分类，分别是'角色a'和'角色b'，其中'角色a'下一级分类有两个子类，分别为'服装1'和'服装2'，'角色b'下则只有一个二级分类，为'服装1'
            内容层级数量不固定，但是至少会有一级类型
    2.3：[game_dir]，游戏使用的mod文件根目录，该目录下对应的都是已激活的mod
        其结构及说明如下：
            [game_dir]，游戏所使用的mod的根目录
                |-[mod_name]，具体mod的文件夹，文件夹名称为[mod_name]，可有多个，不可重名，且每个一级分类也只能有一个
                    |-[预览图]，所在子文件夹对应的mod的预览图，可以有多张
                    |-[mod文件]，所在子文件夹对应的mod的具体mod文件，可以有多个
        文件名规则与[mod_dir]一致
    2.4：mod清单文件，文件名为[mod_list].txt，其中[mod_list]为自定义名称
        位置：软件同目录下
        内容为n行，每一行为一个[mod_name]

三、重要的数据组织行为
    3.1：[mod_list]，完整mod清单，包含了[mod_dir]和[game_dir]下所有mod条目，并标记是否激活，以及计划的操作等信息，具体数据结构如下：
        [mod_list]
            |-[mod一级分类名]，根据所有mod的名称聚类后得到的独特的一级分类名
                |-[mod_name]，如果该分类只有这一个一级实例，则表示该mod的[mod_name]，否则该值为空''
                |-[is_activated]，[mod_name]不为空时有意义，如果该mod位于[game_dir]，则激活标识为true，否则为false
                |-[一级分类下的二级分类名]
                    |-[mod_name]，如果该分类只有这一个二级实例，则表示该mod的[mod_name]，否则该值为空''
                    |-[is_activated]，[mod_name]不为空时有意义，如果该mod位于[game_dir]，则激活标识为true，否则为false
                    |-[二级分类下的三级分类名]
                        |-[mod_name]，如果该分类只有这一个三级实例，则表示该mod的[mod_name]，否则该值为空''
                        |-[is_activated]，[mod_name]不为空时有意义，如果该mod位于[game_dir]，则激活标识为true，否则为false
                        |-[下一级的分类名]，按照前述分类规则递推
                |-[已激活mod_name]，该一级分类下已经激活的[mod_name]，若该一级分类下没有mod被激活，则为空''
